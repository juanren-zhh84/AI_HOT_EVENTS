# 计划.md 整理方案

## Summary
在 `doc/计划.md` 新增一份开发计划文档，把当前项目从“基础服务已连库”推进到“GitHub 采集、快照、热点计算、邮件日报”的实施路径整理清楚。文档使用 UTF-8 Markdown，和现有 `需求文档.md`、`接口文档.md`、`数据库建表语句.sql` 放在同一目录。

## Key Changes
- 新增 `doc/计划.md`。
- 文档结构固定为：
  - 当前状态
  - 开发阶段总览
  - 下一步优先任务
  - 每阶段交付物
  - 验收标准
  - 注意事项
- 计划内容采用以下顺序：
  1. 项目卫生：确保 `.env` 不提交，保留 `.env.example` 思路。
  2. ORM 模型：先实现 `repositories`、`star_snapshots`、`jobs`。
  3. GitHubClient：封装仓库详情、languages、README、rate_limit。
  4. 仓库接口：新增、列表、详情、启停监控。
  5. 星标快照：手动触发采集并写入 `star_snapshots`。
  6. 热点计算：计算 24h/7d 增长并写入 `hot_projects`。
  7. 邮件日报：最后实现模板、订阅者、SMTP 发送和定时任务。

## Test Plan
- 确认 `doc/计划.md` 文件存在。
- 确认 Markdown 中文正常显示，无乱码。
- 确认文档不包含真实 `.env` 密钥、GitHub Token、SMTP 密码。
- 确认计划与当前代码状态一致：当前已有 FastAPI 入口、配置读取、MySQL health check、requirements。
- 确认计划与 `需求文档.md`、`接口文档.md`、`数据库建表语句.sql` 不冲突。

## Assumptions
- 计划文件路径使用 `doc/计划.md`。
- 本地开发环境使用用户自己的 Conda 环境。
