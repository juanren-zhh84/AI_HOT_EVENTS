# GitHub 热点项目自动化监控服务接口文档

版本：v0.1  
日期：2026-06-01  
技术语言：Python  
建议实现：FastAPI + Pydantic + SQLAlchemy

## 1. 接口约定

### 1.1 基础信息

| 项 | 值 |
| --- | --- |
| Base URL | `/api/v1` |
| Content-Type | `application/json; charset=utf-8` |
| 鉴权方式 | `Authorization: Bearer <API_AUTH_TOKEN>` |
| 时间格式 | ISO 8601，默认 UTC 存储，对外可按 `Asia/Shanghai` 展示 |
| 文档地址 | `/docs`、`/openapi.json` |

FastAPI 实现时，接口应使用 Pydantic 模型声明请求体与响应体，并通过 `response_model`、`status_code`、`responses` 等参数生成 OpenAPI 文档。

### 1.2 通用响应结构

成功响应：

```json
{
  "data": {},
  "request_id": "req_01HZY7G9QYAAZ7W3A2X5A0A1B2"
}
```

列表响应：

```json
{
  "data": [],
  "pagination": {
    "page": 1,
    "page_size": 20,
    "total": 100
  },
  "request_id": "req_01HZY7G9QYAAZ7W3A2X5A0A1B2"
}
```

错误响应：

```json
{
  "error": {
    "code": "REPOSITORY_NOT_FOUND",
    "message": "Repository not found",
    "details": {}
  },
  "request_id": "req_01HZY7G9QYAAZ7W3A2X5A0A1B2"
}
```

### 1.3 通用状态码

| 状态码 | 说明 |
| --- | --- |
| 200 | 请求成功 |
| 201 | 创建成功 |
| 202 | 异步任务已创建 |
| 204 | 删除成功，无响应体 |
| 400 | 请求参数错误 |
| 401 | 未认证或 Token 无效 |
| 403 | 无权限或上游限流 |
| 404 | 资源不存在 |
| 409 | 资源冲突 |
| 422 | 请求体验证失败 |
| 429 | 请求过于频繁 |
| 500 | 服务内部错误 |
| 502 | 上游 GitHub 或邮件服务异常 |

### 1.4 枚举

任务状态：

| 值 | 说明 |
| --- | --- |
| `pending` | 等待执行 |
| `running` | 执行中 |
| `succeeded` | 执行成功 |
| `failed` | 执行失败 |
| `cancelled` | 已取消 |

报告状态：

| 值 | 说明 |
| --- | --- |
| `draft` | 草稿 |
| `sending` | 发送中 |
| `sent` | 已发送 |
| `failed` | 发送失败 |

订阅者状态：

| 值 | 说明 |
| --- | --- |
| `active` | 正常订阅 |
| `paused` | 暂停接收 |
| `unsubscribed` | 已退订 |

## 3. 仓库接口

### 3.1 创建监控仓库

`POST /api/v1/repositories`

请求体：

```json
{
  "full_name": "openai/openai-python",
  "source": "manual",
  "tags": ["ai", "sdk"],
  "enabled": true
}
```

字段说明：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `full_name` | string | 是 | GitHub 仓库全名，格式为 `owner/repo` |
| `source` | string | 否 | 来源，默认 `manual` |
| `tags` | string[] | 否 | 本地标签 |
| `enabled` | boolean | 否 | 是否启用监控，默认 true |

响应 `201`：

```json
{
  "data": {
    "id": "repo_01HZY8A3HR9B8Z4S1Y9X7Y6K1M",
    "full_name": "openai/openai-python",
    "html_url": "https://github.com/openai/openai-python",
    "enabled": true,
    "created_at": "2026-06-01T01:00:00Z"
  },
  "request_id": "req_01HZY7G9QYAAZ7W3A2X5A0A1B2"
}
```

### 3.2 查询仓库列表

`GET /api/v1/repositories`

查询参数：

| 参数 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `page` | integer | 1 | 页码 |
| `page_size` | integer | 20 | 每页数量，最大 100 |
| `keyword` | string | - | 按仓库名或描述搜索 |
| `language` | string | - | 主语言过滤 |
| `enabled` | boolean | - | 是否启用 |
| `sort` | string | `stars_desc` | 排序字段 |

响应 `200`：

```json
{
  "data": [
    {
      "id": "repo_01HZY8A3HR9B8Z4S1Y9X7Y6K1M",
      "full_name": "openai/openai-python",
      "description": "The official Python library for the OpenAI API",
      "html_url": "https://github.com/openai/openai-python",
      "primary_language": "Python",

      "forks": 3600,

    "page_size": 20,
    "total": 1
  },
  "request_id": "req_01HZY7G9QYAAZ7W3A2X5A0A1B2"
}
```

## 13. 外部 GitHub API 依赖

服务端应封装 GitHub API 客户端，不直接把 GitHub API 暴露给前端或订阅者。

### 13.1 请求头

```http
Accept: application/vnd.github+json
Authorization: Bearer <GITHUB_TOKEN>
X-GitHub-Api-Version: 2026-03-10
```

如需获取 stargazer 的加星时间，可对 stargazers 接口使用 GitHub Star 自定义媒体类型。

### 13.2 主要上游接口

| 用途 | GitHub REST API |
| --- | --- |
| 查询仓库详情 | `GET /repos/{owner}/{repo}` |
| 查询仓库 Stargazers | `GET /repos/{owner}/{repo}/stargazers?per_page=100&page=1` |
| 查询语言占比 | `GET /repos/{owner}/{repo}/languages` |
| 查询 README | `GET /repos/{owner}/{repo}/readme` |
| 查询目录内容 | `GET /repos/{owner}/{repo}/contents/{path}` |
| 搜索仓库 | `GET /search/repositories?q={query}&sort=stars&order=desc` |
| 查询 API 限流 | `GET /rate_limit` |

### 13.3 上游异常处理

| GitHub 状态 | 处理策略 |
| --- | --- |
| 304 | 使用本地缓存，不计入数据更新 |
| 401 | 标记配置错误，停止相关任务 |
| 403 | 检查限流；限流时延后执行 |
| 404 | 标记仓库不可访问 |
| 422 | 搜索条件或参数无效，记录任务失败 |
| 429 | 延迟重试 |
| 5xx | 指数退避重试 |

## 14. Webhook 与回调预留

MVP 阶段不要求 GitHub Webhook。后续如需更实时的数据，可增加：

| 接口 | 说明 |
| --- | --- |
| `POST /api/v1/webhooks/github` | 接收 GitHub 仓库事件 |
| `POST /api/v1/webhooks/email-events` | 接收邮件服务商投递、打开、退信事件 |
